# Data Model: Maximize Test Coverage

**Feature**: `001.5-maximize-test-coverage`
**Date**: 2026-04-22

## Overview

This sprint introduces **no new database tables or schema changes**. All
existing entities (`conversations`, `messages`, `personas`, `providerConfigs`)
remain unchanged.

The only "data artifact" produced by this feature is the **Coverage Report** —
a set of files generated at test runtime, gitignored, and never persisted to
the database.

---

## New File Artifacts

### Coverage Report

Generated by `@vitest/coverage-v8` into `coverage/` when running
`npm test -- --coverage`. Gitignored.

| File | Description |
|------|-------------|
| `coverage/index.html` | Human-readable HTML report with per-file drilldown |
| `coverage/coverage-summary.json` | Machine-readable JSON summary (used by CI badge tools) |
| `coverage/lcov.info` | LCOV format (optional; for IDE coverage gutters) |

### New Source File

| Path | Description |
|------|-------------|
| `src/app/api/health/route.ts` | New `GET /api/health` endpoint returning `{ status: "ok" }` |
| `.github/workflows/ci.yml` | GitHub Actions workflow running lint + test + coverage |

## Existing Schema (unchanged reference)

No changes to Drizzle schema in `src/modules/shared/db/schema.ts`.

```text
conversations  — id, title, status, personaId, providerId, createdAt, updatedAt
messages       — id, conversationId (FK cascade), role, content, createdAt
personas       — id, name, description, systemPrompt, sortOrder
providerConfigs — id, providerName, modelId, displayName, isAvailable, sortOrder
```
